SECTION 4: ARCHITECTURE
==========================================================

Pattern: Monorepo with three independently deployable sub-applications.
         Backend follows an MVC-like layered architecture.

OVERALL FLOW:
  Browser/Desktop -> React SPA (Vercel)
                  -> Axios (withCredentials, HttpOnly JWT cookie)
                  -> Express REST API (Node.js backend)
                  -> MongoDB Atlas (Mongoose ODM)
                  -> Google Gemini API (AI generation)
                  -> Resync AI Service (external, roadmap + video recommendations)
                  -> Python subprocess (YouTube transcript)
                  -> Cloudinary (avatar uploads)
                  -> Brevo/Mailtrap (email)

BACKEND LAYERS:
  1. Routes      - Define HTTP endpoints, apply middleware
  2. Middlewares - authMiddleware (JWT), multer (file upload)
  3. Controllers - Handle req/res, orchestrate service calls
  4. Services    - Pure business logic (Gemini, EXP, email, actions)
  5. Models      - Mongoose schemas (data persistence)
  6. Prompts     - Gemini prompt templates

FRONTEND LAYERS:
  1. main.jsx     - Router with Public and Dashboard (protected) route groups
  2. Layouts      - PublicLayout / DashboardLayout (sidebar + navbar)
  3. Pages        - Feature-level views
  4. Components   - Shared, reusable UI elements
  5. Api.js       - Centralized Axios instance

GAMIFICATION SYSTEM:
  - EXP formula      : gainedExp = floor(baseExp * (1 + min(streak * 0.05, 1)))
  - Level-up formula : expRequired(level) = floor(300 * level^1.6), max level 100
  - Streak tracked daily; resets if user misses a day

SECTION 11: BUSINESS LOGIC SUMMARY
==========================================================

Resync is a self-directed learning platform with AI-generated structure.

CORE FLOW:
  1. User registers and verifies email.
  2. User creates a Goal (e.g., "Learn React", category: skill/exam/career).
  3. Backend calls Resync AI microservice which uses LLM + YouTube discovery
     to return a structured roadmap (ordered steps) + curated video resources.
  4. User selects the goal with a pace preference (slow/medium/fast).
  5. The system assigns the first LearningStep to the user (UserActiveStep).
  6. Gemini generates daily Actions for that step (micro-tasks with estimated
     time and difficulty level).
  7. User works through Actions day-by-day using a built-in timer:
     - Start session -> Stop session (minutes logged)
     - When minimum required minutes met, action can be marked Complete.
     - Completing an action grants EXP (boosted by streak multiplier).
  8. When all actions for a step are done, user completes the Step.
     For "project" type steps, user must submit a GitHub repo link.
  9. System advances to the next step; level-up occurs when EXP threshold met.
 10. VideoStudy feature: user pastes a YouTube URL, transcript is extracted
     via Python, Gemini generates an MCQ quiz, results are stored for
     weakness analysis.
 11. Dashboard shows: level, EXP, streak, active goal/step/action, progress.
